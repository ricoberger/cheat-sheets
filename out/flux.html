Overview

  • Flux is a tool for keeping Kubernetes clusters in sync with sources of configuration (like Git repositories), and automating updates to configuration when there is new code to deploy.

    Flux is built from the ground up to use Kubernetes' API extension system, and to integrate with Prometheus and other core components of the Kubernetes ecosystem. Flux supports multi-tenancy and support for syncing an arbitrary number of Git repositories.

flux

  • flux reconcile kustomization <NAMESPACE> --with-source - Reconcile Kustomization
  • flux reconcile helmrelease -n <NAMESPACE> <NAME> --with-source --reset --force - Reconcile HelmRelease
  • flux suspend kustomization <NAMESPACE> -n flux-system - Suspend Kustomization
  • flux resume kustomization <NAMESPACE> -n flux-system - Resume Kustomization

kubectl

  • kubectl -n <NAMESPACE> patch helmrelease <HELMRELEASE> --type=json -p='[{"op": "add", "path": "/spec/upgrade", "value": {"remediation": {"retries": 1}} }]' - Patch HelmRelease if it is stuck
  • kubectl -n <NAMESPACE> patch helmrelease <HELMRELEASE> --type=json -p='[{"op": "remove", "path": "/spec/upgrade/remediation"}]' - Remove the patch from HelmRelease once it is fixed
